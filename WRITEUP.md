# WRITEUP — cupflag challenge

### V1 — lvl1.cupflag.top

Простая авторизация по HTTP. Защит и блокировок от источника нет.
В данном случае можно жестко парсить источник в множество воркеров.
Нет необходимости в ротации прокси и рандомных паузах.
В реальных проектах такой подход всегда приносил проблемы и блокировку от источника.

### V2 — lvl2.cupflag.top

Почти аналогичный подход как в V1, но эндпоинт требует `captcha_token`.
Также имеется ограничение на количество запросов.
- Генерируемый токен от claudflare никак не валидируется на ендпоинте.
- Поэтому ендпоинт принимает любую строку и отдает данные в любом случае.
- Т.к. токен никак не проверяется, принято решение не реализовывать его получение из headless браузера.
- Ради интереса было добавлено несколько воркеров с рандомными паузами, для более эффективного поиска флага.

### V3 — lvl3.cupflag.top
- Весь процесс парсинга работает на headless браузере, т.к. источник использует Cloudflare.
- В рамках тестового задания не реализовал использование сессии браузера для передачи её в http клиент.
Это могло бы ускорить получение флага.

## Обнаруженные уязвимости
v1 - просто парсится, никак не защищён
V2 - токен не проверяется и может быть любым
V3 - полученный токен для енпоинта можно использовать в обход claudflare: https://lvl3.cupflag.top/v1/queue/wait?queue_token=token
